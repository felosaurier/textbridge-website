Accessibility Statement

Last updated: February 5, 2026

Our Commitment

TextBridge is committed to ensuring digital accessibility for people with disabilities. We are continually improving the user experience for everyone and applying the relevant accessibility standards.

Conformance Status

This website aims to conform to the Web Content Accessibility Guidelines (WCAG) 2.1 at the AA level. These guidelines explain how to make web content more accessible for people with disabilities and user-friendly for everyone.

Accessibility Features

Our website includes the following accessibility features:

  • Clear and consistent navigation structure
  • Semantic HTML for proper content structure
  • Alternative text for images
  • Keyboard navigation support
  • Skip to main content link
  • Sufficient color contrast ratios
  • Responsive design for various screen sizes
  • ARIA labels for screen readers
  • Focus indicators for interactive elements
  • Reduced motion support for users with vestibular disorders

Technical Specifications

The accessibility of this website relies on the following technologies:

  • HTML5
  • CSS3
  • JavaScript (with graceful degradation)
  • WAI-ARIA

Known Limitations

Despite our best efforts, there may be some limitations. We are aware of the following:

  • Some third-party content may not be fully accessible
  • Older browser versions may not fully support all accessibility features

We are actively working to address these limitations.

Feedback

We welcome your feedback on the accessibility of this website. If you encounter any accessibility barriers, please contact us:

  • Email: contact@textbridge.example
  • Contact Form: Contact Page

We aim to respond to accessibility feedback within 5 business days.

Compatibility

This website is designed to be compatible with:

  • Modern web browsers (Chrome, Firefox, Safari, Edge)
  • Screen readers (JAWS, NVDA, VoiceOver)
  • Keyboard-only navigation
  • Browser zoom up to 200%

Assessment Approach

TextBridge assessed the accessibility of this website through:

  • Self-evaluation
  • Automated accessibility testing tools
  • Manual testing with assistive technologies
  • User feedback from individuals with disabilities

Formal Complaints

If you are not satisfied with our response to your accessibility feedback, you may escalate your concern by contacting our project mentor, Dipl.-Ing. Mag. Dr. Martin Hasitschka, through our contact form.
